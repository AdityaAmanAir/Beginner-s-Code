Data Flow Description

The Data Flow Diagram illustrates how file storage requests are processed in both allocation techniques:

  • User Program initiates file requests (create, read, delete)
  • File System Controller receives and processes the requests
  • Allocator implements the specific storage technique (contiguous or linked-list)
  • Disk Storage is where the files are physically stored based on the allocation method

In contiguous allocation, files are stored in consecutive blocks, while linked-list allocation uses pointers to connect non-consecutive blocks.

Key Differences Between Techniques

  • Contiguous Allocation provides faster access but can lead to fragmentation
  • Linked-list Allocation avoids fragmentation but has overhead for pointer storage
  • Contiguous is better for sequential access patterns
  • Linked-list allows for dynamic file growth without reorganization
